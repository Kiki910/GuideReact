Cos'è l'XML, com'è nato e quali sono le sue principali caratteristiche?

L'XML è un linguaggio che permette la rappresentazione di documenti e dati su supporto digitale.
Sviluppato dal World Wide Web Consortium, nel 1996 è stato formato un gruppo di lavoro oer definire un linguaggio a markup per uso generale. Deriva da SGML, che è un linguaggio di mark-up dichiarativo dell ISO (International Standardization Organization).
L'XML è un linguaggio a marcatori (markup), cioè composto da istruzioni, chiamati tag o marcatori, ch edescrivono la struttura e la forma di un documento; ogni marcatore identifica un elemento o un componente del documento.


Le sue principali caratteristiche del formato XML sono:

  • è facilmente leggibile da qualunque persona;
  • è indipendenet dal tipo di piattaformm ahardware e software su cui viene utilizzato;
  • permette la rappresentazione di qualsiasi tipo di documento e di struttura;
  • è uno standard di pubblico dominio;
  • una applicazione in grado di elaborare dati in formato XML viene definita elaboratore XML.





-Linguaggi e grammatiche

Per stabilire quali tag possono essere utilizzati e come li devono essere usati, ci deve essere una grammatica, cioè un insieme di regole che indica quali elementi possono essere utilizzati e con quale struttura devono essere strutturati i documenti.
Se un documento XML rispetta le regole definite dalla grammatica, si dice che è valido.

Ci sono due tipi di regole:

  • regole sintattiche (o metasintattiche) che definiscono come si devono scrivere le informazioni all'interno dei documenti; se un documento rispetta queste regole, si dice che il documento è ben formato (well-formed);
  • regole semantiche che definiscono cosa si può scrivere all'interno del documento XML; se un documento rispetta sia le regole sintattiche sia quelle semantiche, si dice che il documento è valido.





-Struttura di un documento XML

Un documento XML è strutturato in modo gerarchico. È formato da elementi, ossia dei componenti logici del documento che può contenere una parte di testo oppure altri elementi, che in questo si chiamano "sotto-elementi". Ad ogni elemento possono essere associati degli attributi che ne descrivono delle caratteristiche o delle proprietà; questi attributi sono formati da un nome a cui viene associato un valore.
Gli elementi sono rappresentati attraverso dei tag che racchiudono il contenuto dell'elemento, mentre i sotto-elementi sonodei tag contenuti in altri tag.

Un documento XML è formato da due parti:

  • prologo: contiene una dichiarazione XML (che contiene informazione sulla versione e sul set di caratteri) e occasionalmente un riferimento ad altri documenti che definiscono la struttura;
  • corpo: è il documento XML vero e proprio.

                        
        <?xml version="1.0" encoding="UTF-8"?>    
                                    
        <root>
            <child>
                ...
            </child>
            <child>
                ...
            </child>
        </root>



Il DTD (Document Type Definition)

Il DTD, ovvero Il Document Type Definition, è un liguaggio, o meglio, uno strumento di validazione, per definire la grammatica che descrive i elementi presenti in un documento XML. Per applicare un DTD a un documento XML nel prologo bisogna inserire questa dichiarazione: 

                                
        <!DOCTYPE message SYSTEM "message.dtd">
Un documento DTD è formato da un elenco di dichiarazioni che descrivono la struttura del documento. Queste dichiarazioni definiscono:
  • gli elementi di un documento XML;
  • il modello di contenuto di ogni elemento, cioè gli elementi che può contenere e le sue relazioni;
  • gli attributi associati a ogni elemento, il loro tipo e il loro valore.




-Struttura di un DTD
Per dichiarare un elemento si utilizza la parola chiave ELEMENT che, dopo di esso, ha bisogno del nome dell'elemento stesso e una serie di parametri che verranno spiegati di seguito.
Invece, per dichiarare un attributo dell'elemento, si utilizza la parola chiave ATTLIST che, dopo di esso, ha bisogno del nome dell'attributo e una serie di parametri che verrannno anche spiegati di seguito. 

                                    
    <!ELEMENT catalogo (film*)>
In questo esempio, abbiamo la dichiarazione dell'elemento "catalogo" che può contenere al suo interno 0 o più elementi. Questo si può capire dal simbolo presente dopo la parola film e abbiamo 3 posssibilità:
  1. ? : zero o 1;
  2. + : 1 o più;
  3. * : zero o più.
                                    
    <!ELEMENT formato EMPTY>
    <!ATTLIST formato value (VHS | DVD) #REQUIRED>
In questo altro esempio, abbiamo un dichiarazione dell'elemento "formato"; a differenza di quella di "catalogo", troviamo la parola EMPTY, cioè indica l'elemento vuoto. Al suo posto possiamo trovare la parola ANY, cioè indica che si può solo inserire testo o elementi, ovviamente dichiarati nel DTD.
Dopo alla dichiarazione dell'elemento, abbiamo anche la dichiarazione dei suoi attributi, dove troviamo che i valori possibili sono solo due.
In fondo troviamo la parola "#REQUIRED", cioè l'attributo deve essere presente; altra parola possibile in fondo può essere "#INPLIED", cioè l'attributo è opzionale. Un ulteriore cosa è quello di trovare in fondo la parola #FIXED; questo si usa quando un attributo deve avere un valore prefissato.

                                    
    <!ELEMENT originale EMPTY>
    <!ATTLIST originale value (si | no) "si">
In questo ultimo esempio, abbiamo una dichiarazione dell'elemento "originale", identica a quella di "formato".
Dopo di che abbiamo una dichiarazione di un attributo dell'elemento di sopra, che ha di nuovo due valori possibili. Un particolare da notare è il valore alla fine riportato tra virgolette che indica il valore di default.

Il metodo fetch
La Fetch è un metodo che, come per il JSON, ci permette di mandare una richiesta al server e caricare informazioni ogni volta che ne abbiamo bisogno. A differenza di quanto avviene con il JSON, per l'XML, prima din utilizzare concretamente i dati, serviranno dei passaggi in più.

Per usare la fetch servono 3 "parole chiavi":
  • fetch(URL): ha bisogno dell'URL da raggiungere per estrapolare le informazioni;
  • .then(): il primo then() serve per trasformare le informazioni prese tramite l'URL e poi convertirlo in text;
  • .then(): il secondo then() serve per prendere i suoi dati./li>

Oltre alla fetch, che in poche parole estrapola solamente le informazioni dal server, abbiamo bisogno del DOMParser, un interfaccia che ci permette di parsare codice XML da una stringa. 

                        
        const carica=()=>{
            fetch("https://raw.githubusercontent.com/icobasco/sample_data_files/master/4C_catalogoFilm.xml")       
            .then((catalogo) => catalogo.text())
            .then((datixml) => {
                let mioXML = new DOMParser();
                let parseXML = mioXML.parseFromString(datixml, "text/xml")
                let titoli = (parseXML.querySelectorAll("titolo"))
                let genere = (parseXML.querySelectorAll("genere"))
                let anno = (parseXML.querySelectorAll("anno"))
                for (let i = 0; i < titoli.length; i++) {
                    let elemento1 = document.createElement("div")
                    elemento1.innerHTML = titoli[i].innerHTML
                    document.getElementById("title").appendChild(elemento1)
                }
                for (let i = 0; i < titoli.length; i++) {
                    let elemento2 = document.createElement("div")
                    elemento2.innerHTML = genere[i].attributes.value.textContent
                    document.getElementById("genere").appendChild(elemento2)
                }
                for (let i = 0; i < titoli.length; i++) {
                    let elemento3 = document.createElement("div")
                    elemento3.innerHTML = anno[i].attributes.value.textContent
                    document.getElementById("anno").appendChild(elemento3)
                }
            })
        }
